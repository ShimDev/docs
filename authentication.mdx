# Authentication

All Shim API requests require an API key passed in the `Authorization` header.

---

## Getting Your API Key

<Steps>
  <Step title="Sign up">
    Create a free account at [shim.so/signup](https://shim.so/signup)
  </Step>
  <Step title="Navigate to Dashboard">
    Go to [shim.so/dashboard](https://shim.so/dashboard)
  </Step>
  <Step title="Generate Key">
    Click "API Keys" â†’ "Generate New Key"
  </Step>
  <Step title="Copy and Store">
    Copy the key immediately. It won't be shown again.
  </Step>
</Steps>

---

## Using Your API Key

Include your API key in the `Authorization` header as a Bearer token:

```bash
Authorization: Bearer sk_live_xxxxx
```

### Example Request

```typescript
const response = await fetch('https://api.shim.so/v1/repair', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer sk_live_xxxxx',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ raw_output: '{"broken": "json"' })
});
```

## Key Format

- **Production keys:** `sk_live_xxxxx`
- **Test keys:** `sk_test_xxxxx` (coming soon)

<Warning>
Never commit API keys to version control. Use environment variables:
```bash
export SHIM_API_KEY=sk_live_xxxxx
```
</Warning>

## Security Best Practices

<AccordionGroup>
  <Accordion title="Rotate keys regularly">
    Generate a new key every 90 days. Delete old keys immediately.
  </Accordion>
  <Accordion title="Use environment variables">
    Never hardcode keys in source code. Use `.env` files (gitignored).
  </Accordion>
  <Accordion title="Restrict key scope">
    Create separate keys for development, staging, and production.
  </Accordion>
  <Accordion title="Monitor usage">
    Check your dashboard for unexpected spikes in usage.
  </Accordion>
</AccordionGroup>

## Troubleshooting

### 401 Unauthorized

**Cause:** Invalid or missing API key  
**Fix:** Check that your key is correct and included in the `Authorization` header

### 429 Too Many Requests

**Cause:** Rate limit exceeded  
**Fix:** Implement exponential backoff or upgrade your tier

See [Troubleshooting](/troubleshooting) for more common issues.
